# macintosh os

## 프로젝트 설정

### Vite 설정

- Vite 설치 명령어 : `npm create vite@latest`
- Vite 선정 이유
  - React 프로젝트 설정 방식에는 `Vite`와 `CRA`(`Create React App`) 방식이 존재했다 하지만 CRA 방식은 최근 사용하지 않는다고 하며 Vite를 사용하여 빠르게 웹 개발 시작이 가능하다고 하여 Vite로 선정하였다.
  - `Vite` : 프론트엔드 빌드 도구

### tailwind 설정

- tailwindcss 설치 명령어 : `npm install -D tailwindcss@^3 postcss autoprefixer`
- `postcss.config.js` 설정하기

  ```js
  export default {
    plugins: {
      tailwindcss: {},
      autoprefixer: {},
    },
  };
  ```

- `tailwind.config.js` 설정하기

  ```js
  /** @type {import('tailwindcss').Config} */
  export default {
    content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx}"],
    theme: {
      extend: {},
    },
    plugins: [],
  };
  ```

- tailwind 선정이유

  - 기존에는 단순 CSS로만 개발했었는데 이번 기회에 tailwind를 경험해보고 싶은 이유가 가장 컸다.
  - tailwind를 사용하면 레고 조립하는 느낌으로 작성할 수 있다고 해서 굉장히 흥미로웠다.

- tailwind에서 v3로 낮춘 이유
  - `npm install -D tailwindcss postcss autoprefixer`를 통해 설치를 진행했는데 설치가 되지 않는 오류가 존재했다.
  - v3로 다운그레이드 해서 파일을 직접 생성하니 원활히 동작하는 것을 확인했고 이를 토대로 개발하면 좋을 거 같아서 바로 작성하게 되었습니다.

### Zustand 설정

- Zustand 설치 명령어 : `npm install zustand`
- Zustand 선정 이유
  - Window의 상태(열고 닫고 최소화 등)를 관리하기 위해서는 상태 관리가 불가피 했다.
  - 고민해보던 도중에 다양한 라이브러리를 경험해보고자하는 마음으로 Context API나 Props Drilling을 경험하는 대신 Zustand라는 라이브러리 도입을 선택하게 되었고 사용해보게 되었다!
  - 해당 과정에서 많은 시간이 소비된거 같은데 figma로 컴포넌트 구성을 계속 해서 짜고 수정을 반복했고 이 해결책으로 Zustand를 선정한 것도 이유가 있었다.(원하는 곳에 구독할 수 있다 라는 장점이 매우 컸던거 같다.)
